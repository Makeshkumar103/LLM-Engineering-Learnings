# Week 6 Architecture

## Overview

Week 6 builds a product price prediction system ("The Price is Right"). It progresses through data curation from Amazon Reviews, traditional ML baselines, deep neural networks, zero-shot LLM pricing, and GPT fine-tuning. The `pricer/` package is the core reusable module.

---

## Core Package: `pricer/`

### `pricer/items.py` — Data model

**`class Item`** (Pydantic `BaseModel`):
- Fields: `title`, `category`, `price`, `full`, `weight`, `summary`, `prompt`, `completion`
- Source: Amazon Reviews 2023 dataset (HuggingFace: `McAuley-Lab/AmazonReviews2023`)
- `make_prompts(tokenizer, max_tokens, do_round)` — truncates summary, creates prompt/completion pairs
- `from_hub()` / `push_to_hub()` — HuggingFace dataset loading/saving

### `pricer/evaluator.py` — Evaluation engine

**`class Tester`:**
- `predictor` function, test data, multi-threaded eval via `ThreadPoolExecutor` (5 workers)
- `post_process(value)` — regex price extraction from LLM output
- `run()` → `report()` → scatter plot (predicted vs actual) + cumulative error trend (95% CI)

### Module Structure

```
pricer/
├── __init__.py
├── items.py                # Item data model, prompt construction, HF dataset I/O

```

## Data Flow

### 1. Data Curation

```
Amazon Reviews 2023 (HuggingFace)
  → Filter: RAW_grocery category only
  → ~800K train + 10K val + 10K test
  → Item objects with title, category, price, summary
  → Summaries generated by LLM (Groq via LiteLLM)
```

### 2. ML Baselines

```
Features: HashingVectorizer (n=5000) on title + category + summary

Models trained:
  ├── Constant (median price) → $106.18 error
  ├── Linear Regression      → $101.56
  ├── NLP + Linear Regression→ $76.81
  ├── Random Forest          → $72.28
  └── XGBoost                → $68.23
```

### 3. Deep Neural Network

```
Input: HashingVectorizer (5000-dim) → log(price) normalization
  10-layer Residual Network (4096 hidden dim):
    Linear → LayerNorm → ReLU → Dropout (×10 with skip connections)
  Output: 1 scalar → exp(pred * Y_STD + Y_MEAN) - 1
  
  Training: AdamW, MSE loss, batch size 512
  Result: $46.49 avg error
```

### 4. Zero-Shot LLM Pricing

```
Prompt: "Estimate the price of this product: {summary}. Price is $"
Models tested:
  ├── GPT 4.1 Nano → $62.51
  ├── Grok 4.1 Fast → $57.62
  ├── Gemini 3 Pro → $50.54
  ├── Claude 4.5 Sonnet → $47.10
  └── GPT 5.1 → $44.74 (best)
```

### 5. GPT Fine-Tuning

```
Prompt/completion pairs → OpenAI fine-tuning API
  Model: gpt-4.1-nano
  Result: $75.91 avg error (worse than zero-shot, likely due to small model capacity)
```

## Architecture Patterns

### 1. Progressive Model Comparison

Each approach is evaluated on identical test data with the same error metric:

```
Baseline → ML → DNN → Zero-shot LLM → Fine-tuned LLM
                ↓
        All compared in results.ipynb
```

### 2. Deep Residual Network

```python
class ResidualBlock(nn.Module):
    def __init__(self, dim):
        self.net = nn.Sequential(
            nn.Linear(dim, dim),
            nn.LayerNorm(dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(dim, dim),
        )
    def forward(self, x):
        return x + self.net(x)  # skip connection
```

10 blocks stacked, 4096 hidden dimensions, ~151M-289M parameters.

### 3. Log-Normalized Price Regression

```python
log_price = (torch.log(price + 1) - Y_MEAN) / Y_STD  # normalize
prediction = torch.exp(pred * Y_STD + Y_MEAN) - 1     # denormalize
```

Prices are heavy-tailed; log transform makes the distribution more Gaussian for MSE training.

### 4. Multi-Threaded Evaluation

```python
with ThreadPoolExecutor(max_workers=5) as pool:
    results = list(pool.map(self.run_datapoint, range(self.size)))
```

Parallel inference across 5 threads. Each thread: predict → post-process → compute error → color-code (green/orange/red).

## Key Concepts

1. **Data curation** — Filtering, cleaning, and summarizing large datasets with LLMs
2. **HashingVectorizer** — Bag-of-words features for text without embeddings
3. **Residual networks** — Skip connections for training deep regressors
4. **Zero-shot LLM pricing** — Frontier models as domain experts without training
5. **Log-normalized regression** — Handling heavy-tailed price distributions
6. **Model evaluation** — Consistent test set, multi-threaded inference, visualization
7. **Fine-tuning vs. zero-shot** — Smaller fine-tuned models can underperform larger zero-shot ones
